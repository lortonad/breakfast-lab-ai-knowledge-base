# Sample documents

Three documents from **Driftwood**, a fictional Traverse City startup
building booking-and-waiver software for independent outdoor outfitters
(kayak liveries, bike rentals, and the like). Every person, company, and
llama in these files is invented.

They exist so that anyone who arrives at the workshop without documents
of their own — or anyone who'd rather practice on someone else's mess
before shelving their own — has realistic material for the apprentice
test:

- `customer-interview-dale-torch-lake-paddle.md` — a call transcript.
  Practice separating people, companies, and pain points from small
  talk and scheduling noise.
- `strategy-memo-q3-draft.md` — a founder's late-night internal memo.
  Practice separating decisions from open questions, and durable facts
  from things that are only true this quarter.
- `competitor-notes-peakbook.md` — pasted-together research. Practice
  separating verified fact from interpretation from trade-show rumor,
  and noticing which claims have dates.

The three documents deliberately overlap — the same people and problems
appear in more than one — so that processing all three produces a small,
genuinely interlinked wiki rather than three islands.

A note on workflow: this folder is the practice shelf, not the
collection. When the Librarian uses one of these files, it copies it
into `raw/` first, because everything shelved in the wiki must have a
source in the permanent collection. Pages built from practice material
are marked as such, so they're easy to clear when your real documents
arrive.

If this is your own repository and you have real documents: delete this
folder whenever you like. The Librarian will not miss it.
